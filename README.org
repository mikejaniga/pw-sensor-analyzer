#+TITLE: Mine Sensor Measurement Analyzer
#+AUTHOR: Michał Janiga

*** DESCRIPTION
A console tool used to consolidate data from multiple CSV files into a single result matrix. The script processes measurement series from different days, handling a variable number of sensors between readings.

*** ENVIRONMENT AND DEPENDENCIES
The project uses the ~uv~ tool for dependency and environment management. The project configuration is located in the ~pyproject.toml~ file. Python version 3.12 or newer is required. The main packages are ~pandas~ and ~openpyxl~.

*** INSTALLATION
#+begin_src bash
uv sync
#+end_src

To install the tool globally on the system:
#+begin_src bash
uv tool install .
#+end_src

*** USAGE
In the environment:
#+begin_src bash
uv run sensor-analyzer --start DD.MM.YYYY --end DD.MM.YYYY [options]
#+end_src

Globally:
#+begin_src bash
sensor-analyzer --start DD.MM.YYYY --end DD.MM.YYYY [options]
#+end_src

*** ARGUMENTS
- ~-s, --start~ :: start date in DD.MM.YYYY format (required)
- ~-e, --end~ :: end date in DD.MM.YYYY format (required)
- ~-o, --output~ :: output format (console, excel, both - default: both)
- ~-f, --file~ :: target filename for the xlsx result
- ~-d, --dir~ :: directory containing input CSV files (default: sample_data)
- ~--debug~ :: developer view

*** VALIDATION RULES AND MARKINGS
The measurement duration must be at least half the median time of series measurements.
- ~+~ :: correct measurement, no alarm
- ~!~ :: correct measurement, active alarm
- ~-~ :: measurement time too short or sensor missing from file
